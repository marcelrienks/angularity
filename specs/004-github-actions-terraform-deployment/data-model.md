# Data Model: GitHub Actions + Terraform Automated Deployment

**Feature**: 004-github-actions-terraform-deployment  
**Date**: May 1, 2026  
**Purpose**: Document the key entities, relationships, and state transitions for the deployment automation system

---

## Entity Overview

This feature doesn't introduce new data entities; instead, it models the **deployment pipeline state** and **infrastructure configuration state**. The core entities are AWS infrastructure resources and GitHub Actions configuration.

---

## 1. AWS Infrastructure Resources

### S3 Bucket (Static Site Storage)

**Entity**: `aws_s3_bucket.website`

| Field | Type | Required | Validation | Notes |
|-------|------|----------|-----------|-------|
| `bucket_name` | String | Yes | Globally unique, lowercase alphanumeric + hyphens | Auto-generated: `mx5-alignment-{account-id}` |
| `versioning_enabled` | Boolean | No | True/False | Default: `true` (enables rollback) |
| `encryption_enabled` | Boolean | No | True/False | Default: `true` (AES256 at-rest) |
| `public_access_blocked` | Boolean | No | True/False | Default: `false` (CloudFront-only access) |
| `index_document` | String | No | Valid HTML file | Default: `index.html` |
| `error_document` | String | No | Valid HTML file | Default: `index.html` (SPA routing) |
| `creation_date` | DateTime | Auto | ISO 8601 | Terraform-managed |
| `region` | String | Required | AWS region code | Default: `us-east-1` |

**Relationships**:
- 1→1: To CloudFront Distribution (bucket is origin)
- 1→N: To S3 Objects (files deployed to bucket)
- 1→1: To IAM Policy (github-actions has read/write permissions)

**State Transitions**:
```
CREATED → CONFIGURED (ACL + policy set) → POPULATED (files synced) → ACTIVE
   ↓
   └─→ DELETED (terraform destroy)
```

**Validation Rules**:
- Bucket name must be globally unique (AWS enforces)
- Must be in same region as CloudFront origin (typically us-east-1)
- Versioning should be enabled for production safety
- Should have encryption enabled per AWS best practices

### CloudFront Distribution (CDN & Custom Domain)

**Entity**: `aws_cloudfront_distribution.website`

| Field | Type | Required | Validation | Notes |
|-------|------|----------|-----------|-------|
| `distribution_id` | String | Auto | AWS-generated ID | Used for cache invalidation |
| `domain_name` | String | Auto | CloudFront domain (d*.cloudfront.net) | Auto-generated by AWS |
| `origin_domain` | String | Required | S3 bucket domain | Points to S3 origin |
| `custom_domain` | String | No | Valid FQDN | e.g., `alignment.marcelrienks.com` |
| `acm_certificate_arn` | String | Conditional | ARN of ACM certificate | Required if `custom_domain` set |
| `default_ttl_seconds` | Integer | No | Positive integer | Default: 3600 (1 hour) |
| `default_root_object` | String | No | Valid filename | Default: `index.html` |
| `price_class` | String | No | `PriceClass_100|200|All` | Default: `PriceClass_100` |
| `enabled` | Boolean | No | True/False | Default: `true` |
| `creation_date` | DateTime | Auto | ISO 8601 | Terraform-managed |
| `etag` | String | Auto | AWS-generated | Used for versioning |

**Relationships**:
- 1→1: To S3 Bucket (origin)
- 1→1: To ACM Certificate (if custom domain)
- 1→1: To Route53 Record (DNS alias)
- 1→N: To CloudFront Invalidations (cache clear operations)

**State Transitions**:
```
CREATED → ENABLED → ROUTING_TRAFFIC → ACTIVE
   ↓
   └─→ DISABLED (manual) → DELETED
```

**Validation Rules**:
- If `custom_domain` is set, `acm_certificate_arn` must be provided
- ACM certificate must be in `us-east-1` (CloudFront requirement)
- Default TTL should be balanced (too low = extra origin requests; too high = stale content)
- Origin domain must resolve and return 200 OK responses

---

### ACM Certificate (HTTPS for Custom Domain)

**Entity**: `aws_acm_certificate.alignment`

| Field | Type | Required | Validation | Notes |
|-------|------|----------|-----------|-------|
| `certificate_arn` | String | Auto | AWS ARN format | Unique identifier |
| `domain_name` | String | Required | Valid FQDN | `alignment.marcelrienks.com` |
| `validation_method` | String | Required | `DNS` \| `EMAIL` | Default: `DNS` (automated) |
| `validation_status` | String | Auto | `PENDING` \| `SUCCESS` \| `FAILED` | Managed by validation resource |
| `expiration_date` | DateTime | Auto | ISO 8601 | AWS auto-renews for free |
| `region` | String | Required | `us-east-1` | CloudFront requirement |

**Relationships**:
- 1→1: To Route53 Zone (for DNS validation)
- 1→1: To CloudFront Distribution (custom domain HTTPS)

**State Transitions**:
```
CREATED → DNS_VALIDATION_PENDING → VALIDATED → ACTIVE
                    ↓
                    └─→ VALIDATION_FAILED (retry needed)
```

**Validation Rules**:
- Domain must not contain wildcards for this use case (specific subdomain only)
- DNS validation requires Route53 zone write access
- Validation records must be created and left in place for certificate renewal
- Must be in `us-east-1` for CloudFront integration

---

### Route53 DNS Record (Domain Resolution)

**Entity**: `aws_route53_record.alignment`

| Field | Type | Required | Validation | Notes |
|-------|------|----------|-----------|-------|
| `record_name` | String | Required | FQDN | `alignment.marcelrienks.com` |
| `record_type` | String | Required | `A` (alias) | AWS-specific alias type |
| `hosted_zone_id` | String | Required | Valid Route53 zone ID | Zone: `marcelrienks.com` |
| `target_domain` | String | Required | CloudFront domain name | e.g., `d1234abc.cloudfront.net` |
| `target_zone_id` | String | Required | CloudFront zone ID | AWS-managed, varies by region |
| `set_identifier` | String | No | String | For weighted/failover (not used here) |
| `ttl` | Integer | Auto | Alias records have no TTL | N/A for alias records |

**Relationships**:
- 1→1: To Route53 Hosted Zone (parent zone)
- 1→1: To CloudFront Distribution (target)

**State Transitions**:
```
CREATED → DNS_PROPAGATING → ACTIVE → RESOLVING
   ↓
   └─→ DELETED
```

**Validation Rules**:
- Record name must be valid subdomain of parent zone
- Target must be valid CloudFront distribution domain
- Alias records automatically update if CloudFront domain changes

## 2. Deployment Pipeline State

### GitHub Actions Execution

**Entity**: Workflow Execution (implicit, tracked by GitHub)

| Field | Type | Scope | Meaning |
|-------|------|-------|---------|
| `trigger_event` | Enum | Per run | `push` \| `workflow_dispatch` (manual) |
| `branch` | String | Per run | Git branch (only `main` triggers auto-deploy) |
| `test_status` | Enum | Per run | `PENDING` → `RUNNING` → `PASSED` \| `FAILED` |
| `deployment_status` | Enum | Per run | `SKIPPED` (if tests fail) → `RUNNING` → `SUCCESS` \| `FAILED` |
| `timestamp` | DateTime | Per run | UTC timestamp of execution |
| `logs` | Text | Per run | Execution logs (GitHub Actions stores) |

**State Transitions** (per workflow run):
```
QUEUED → RUNNING_TESTS → TESTS_PASSED → RUNNING_DEPLOY → SUCCESS
                ↓
         TESTS_FAILED → DEPLOY_SKIPPED → FAILED
                ↓
         DEPLOY_ERROR → FAILED
```

**Validation Rules**:
- Only `main` branch triggers automatic deployment
- Tests must complete with exit code 0 for deployment to proceed
- Secrets (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`) must be available
- Optional: `CLOUDFRONT_DISTRIBUTION_ID` (deployment works without it, but cache invalidation skipped)

---

### S3 Sync Operation (File Deployment)

**Entity**: S3 Sync Transaction

| Field | Type | Validation | Notes |
|-------|------|-----------|-------|
| `source_directory` | String | Must exist locally | `./site/` |
| `destination_bucket` | String | Must exist in AWS | `mx5-alignment-{account-id}` |
| `files_uploaded` | Integer | > 0 | Number of files synced |
| `files_deleted` | Integer | ≥ 0 | Files removed from S3 |
| `cache_control` | String | Valid HTTP header | `public, max-age=3600` |
| `sync_timestamp` | DateTime | ISO 8601 | When sync completed |
| `status` | Enum | `SUCCESS` \| `FAILED` | Outcome |

**Validation Rules**:
- Source directory must contain at least one file
- Destination bucket must exist and be writable
- `--delete` flag removes old files (enable carefully)
- Cache-Control headers affect CloudFront caching behavior

---

### CloudFront Cache Invalidation

**Entity**: Invalidation Request

| Field | Type | Required | Validation | Notes |
|-------|------|----------|-----------|-------|
| `invalidation_id` | String | Auto | AWS-generated | Unique per invalidation |
| `distribution_id` | String | Required | Valid CloudFront distribution ID | From GitHub secret |
| `paths` | Array[String] | Required | Glob patterns | `/*` (all files) or specific paths |
| `status` | Enum | Auto | `InProgress` \| `Completed` \| `Failed` | AWS-managed |
| `created_time` | DateTime | Auto | ISO 8601 | When invalidation requested |
| `completion_time` | DateTime | Auto | ISO 8601 | When invalidation completed (2-5 min) |

**Relationships**:
- 1→1: To CloudFront Distribution
- 1→1: To S3 Sync Operation (logically sequenced)

**Validation Rules**:
- Distribution ID must be valid CloudFront distribution
- Paths must be valid CloudFront path patterns
- Invalidation is optional (gracefully skipped if distribution ID not available)

## 3. Infrastructure Configuration (terraform.tfvars)

**Entity**: Terraform Variables Configuration

| Variable | Type | Required | Default | Validation | Purpose |
|----------|------|----------|---------|-----------|---------|
| `aws_region` | String | No | `us-east-1` | Valid AWS region | Deployment region |
| `environment` | String | No | `prod` | String | Environment label |
| `project_name` | String | No | `mx5-alignment` | Alphanumeric + hyphens | Resource naming |
| `custom_domain` | String | No | (empty) | Valid FQDN or empty | Custom domain (optional) |
| `acm_certificate_arn` | String | Conditional | (empty) | Valid ARN or empty | Required if custom_domain set |
| `bucket_name` | String | No | (empty) | Alphanumeric + hyphens or empty | Override auto-generated name |
| `cache_ttl_seconds` | Integer | No | `3600` | Positive integer | CloudFront TTL |
| `enable_versioning` | Boolean | No | `true` | True/False | S3 versioning |
| `enable_monitoring_alerts` | Boolean | No | `false` | True/False | CloudWatch alerts |

**File Location**: `infrastructure/terraform.tfvars`

**Creation Requirement**: Must be created before `terraform apply` (not committed to version control due to sensitive values)

**Sample Content**:
```hcl
aws_region       = "us-east-1"
environment      = "prod"
project_name     = "mx5-alignment"
custom_domain    = "alignment.marcelrienks.com"
acm_certificate_arn = "arn:aws:acm:us-east-1:365620267529:certificate/CERT-ID-HERE"
cache_ttl_seconds = 3600
enable_versioning = true
```

## 4. IAM Policy & Permissions

**Entity**: GitHub Actions IAM User Policy

| Permission | Resource | Action | Purpose |
|-----------|----------|--------|---------|
| S3 Deployment | `arn:aws:s3:::mx5-alignment-*` | `s3:PutObject`, `s3:DeleteObject` | Upload/sync site files |
| S3 Deployment | `arn:aws:s3:::mx5-alignment-*` | `s3:GetObject` | Verify uploaded files |
| S3 Listing | `arn:aws:s3:::mx5-alignment-*` | `s3:ListBucket` | List bucket contents |
| CloudFront Cache | `arn:aws:cloudfront::365620267529:distribution/*` | `cloudfront:CreateInvalidation` | Clear cache after deploy |
| CloudFront Read | `*` | `cloudfront:GetDistribution`, `cloudfront:ListDistributions` | Get distribution info |

**User**: `github-actions` (existing IAM user)

**Validation Rules**:
- Policy must be scoped to specific resources (not wildcards)
- Must allow S3 operations only on alignment bucket
- CloudFront operations may use broader resource patterns (no invalidation per-distribution filtering available)

## 5. Terraform State

**Entity**: Terraform State File (`terraform.tfstate`)

| Attribute | Type | Contents | Sensitivity |
|-----------|------|----------|-------------|
| `version` | Integer | Terraform state format version | Low |
| `serial` | Integer | Increment on each apply | Low |
| `terraform_version` | String | Version used to create state | Low |
| `resources` | Array | AWS resource references + attributes | **HIGH** (contains bucket names, ARNs, IDs) |
| `outputs` | Object | Exported values (CloudFront ID, S3 bucket) | **HIGH** (infrastructure metadata) |

**File Location**: `infrastructure/terraform.tfstate` (local) or remote S3 bucket (future)

**Security Notes**:
- File contains sensitive infrastructure metadata
- Should be kept secure (not publicly readable)
- Initial deployment uses local state; team collaboration requires S3 remote backend

## Cross-Entity Validation Rules

| Rule | Entities | Constraint |
|------|----------|-----------|
| **ACM Certificate Region** | ACM, CloudFront | Certificate must be in `us-east-1` |
| **Custom Domain Completeness** | CloudFront, ACM, Route53 | If custom_domain set, acm_certificate_arn AND route53_record required |
| **S3 Bucket Accessibility** | S3, CloudFront, GitHub Actions | CloudFront OAC must have S3 read access |
| **GitHub Secrets Completeness** | GitHub Actions, AWS | AWS_ACCESS_KEY_ID + AWS_SECRET_ACCESS_KEY required; CLOUDFRONT_DISTRIBUTION_ID optional |
| **Terraform Outputs** | Terraform, GitHub Actions, Documentation | CLOUDFRONT_DISTRIBUTION_ID output must be captured and added to GitHub secrets |

## Deployment Success Criteria (Data Integrity)

1. ✅ **S3 Bucket Created**: Bucket name follows pattern `mx5-alignment-{account-id}`
2. ✅ **CloudFront Distribution Active**: Distribution domain resolves and serves S3 content
3. ✅ **ACM Certificate Validated**: Certificate status = ACTIVE for `alignment.marcelrienks.com`
4. ✅ **Route53 Record Active**: DNS resolves `alignment.marcelrienks.com` → CloudFront domain
5. ✅ **IAM Permissions Correct**: `github-actions` user can read/write S3 and create CloudFront invalidations
6. ✅ **GitHub Secrets Available**: Workflow can retrieve AWS credentials and distribution ID
7. ✅ **Sync Idempotency**: Multiple deployments produce identical S3 state (versioning tracks changes)
8. ✅ **Cache Invalidation Functional**: Post-deployment, CloudFront serves latest files within 2 minutes
